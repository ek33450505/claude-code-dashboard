### 2. AgentEntity State Machine — Missing Transition Guard from GATHERING
**File:** `src/engine/AgentEntity.ts:71-108`  
**Severity:** CRITICAL — DEADLOCK RISK

```typescript
// Line 78: Only guards ACTIVE state
if (wasLive && !this.isLive && this.state === 'ACTIVE') 
  this.transitionTo('REACTING')
```

**Problem:**
When liveness goes from true → false while in GATHERING state, no transition occurs. Agent stays in GATHERING forever with no wander fallback.

**Scenario:**
1. Agent becomes ACTIVE → reaches gather point → transitionTo('GATHERING')
2. Liveness check: wasLive=true, isLive=false, state=GATHERING
3. Line 78 condition fails (only checks === 'ACTIVE')
4. Agent never returns to IDLE or REACTING → **DEADLOCK**

**Required Fix:**
```typescript
if (wasLive && !this.isLive && (this.state === 'ACTIVE' || this.state === 'GATHERING')) 
  this.transitionTo('REACTING')
```

### 3. TileRenderer.drawAgents() — Painter's Algorithm Re-Sort Every Frame
**File:** `src/engine/TileRenderer.ts:283-295`  
**Severity:** CRITICAL PERFORMANCE

```typescript
drawAgents(entities: AgentEntity[], camera: Camera) {
  // ⚠️ Creates new array and sorts every single frame
  const sorted = [...entities].sort((a, b) => a.worldY - b.worldY)
  for (const entity of sorted) {
    // ... render ...
  }
}
```

**Problem:**
- `[...entities]` allocates new array every frame (28 agents × 60fps = 1,680 allocations/sec)
- `.sort()` is O(n log n) every frame (expensive even though agents rarely change Y-order)
- Unnecessary GC pressure and CPU burn on sort algorithm

**Impact:** Noticeable frame drops on lower-end devices.

**Required Fix:** Pre-allocate and reuse sorted array:
```typescript
// Class property
private sortedEntities: AgentEntity[] = []

drawAgents(entities: AgentEntity[], camera: Camera) {
  this.sortedEntities.length = entities.length
  for (let i = 0; i < entities.length; i++) {
    this.sortedEntities[i] = entities[i]
  }
  this.sortedEntities.sort((a, b) => a.worldY - b.worldY)
  
  for (const entity of this.sortedEntities) {
    // ... render ...
  }
}
```

### 4. GameLoop.rafId Not Cleared on First Stop
**File:** `src/engine/GameLoop.ts:32-35`  
**Severity:** CRITICAL (low likelihood, high impact)

```typescript
private rafId: number | null = null

start() {
  if (this.running) return
  this.running = true
  this.rafId = requestAnimationFrame(this.tick)  // ← Could overwrite previous ID
}

stop() {
  this.running = false
  if (this.rafId !== null) { 
    cancelAnimationFrame(this.rafId)
    this.rafId = null 
  }
}
```

**Status:** ✓ SAFE — The guards are correct. If stop() is called while running, `this.rafId !== null` check prevents errors.

### 5. Pathfinder.findPath() — O(n²) Open List Search Cost
**File:** `src/engine/Pathfinder.ts:22-23`  
**Severity:** WARNING

```typescript
while (open.length) {
  open.sort((a, b) => a.f - b.f)  // ⚠️ O(n log n) every iteration
  const cur = open.shift()!
  // ... process node ...
}
```

**Problem:**
- Sorts open list every iteration (should use priority queue)
- 28 agents × frequent pathfinding = O(n² log n) operations per session
- With 280-tile rooms, this accumulates during transitions

**Not Critical Yet:** Only runs on state transitions, not every frame. But scales poorly if agent count grows beyond 30.

**Fix:** Replace with binary heap for O(log n) insertion/extraction.

### 12. Canvas DPR Scaling
**File:** `src/components/GameWorld.tsx:68-74`  
**Severity:** SUGGESTION (Low, intentional design choice)

The canvas is sized 1:1 pixel ratio (VIEWPORT_W=960, canvas.width=960). On high-DPI displays (e.g., Retina with DPR=2), canvas will be blurry if CSS scaling to 960px.

**Current Approach:** CSS `imageRendering: pixelated` compensates, which is fine for pixel art.

**Alternative:** Support DPR for crisp rendering on all devices (but requires scale context, more complex).
