How alive should the office be?

Three design directions for the Pokemon-style CAST HQ — click one to select

A — Pixel Room, Rich Desks

Agents stay at fixed desks in a tile-drawn office room. Desk items animate (monitors flicker, coffee steams, keyboards clack). Agent sprites have many more animation frames — typing, sleeping, celebrating, talking.

Lowest complexity

B — Walking Agents

Agents walk freely around the office floor. They move to workstations when assigned tasks, gather at a meeting table when chained together, and wander when idle. Full 4-directional walk cycles.

Medium complexity

C — Full Game World

A scrollable multi-room office built with Phaser.js or a custom tilemap engine. Departments become rooms. Agents travel between rooms. Dialog boxes, interaction zones, and event scripting — exactly like a Pokemon overworld.

Highest complexity
Current approach: AgentOffice.tsx uses DOM-based CSS sprites (80 divs/agent, 28 agents = 2,240 DOM nodes) with Framer Motion physics. A Game Boy–style office would replace or sit alongside this. All three options use HTML5 Canvas for the game world itself.
