Character Per Agent — How Distinct?

Currently 28 agents share 7 archetype templates (same sprite, different colors). How unique should each character be?

A

Same Archetype, More Colors

Keep the 7 archetype body shapes. Each agent is already color-distinct. Fastest to ship — zero new sprites needed.

Pros

  • No sprite work
  • Already done

Cons

  • Agents in same dept look similar
  • Less "character"
B

Unique Sprites Per Agent

28 hand-crafted 8×10 pixel people — each with a role-specific look (tools, clothing, hat style). Same palette system.

Pros

  • Each agent feels like a real character
  • Recognizable by look alone
  • Gallery-worthy

Cons

  • 28 sprites to design
  • Largest scope addition
C

Hybrid — Archetype Variants

Keep 7 archetypes but add 2-3 visual variants within each (different hair, tools, accessories). ~15 sprite designs total.

Pros

  • More variety than current
  • Half the design work of B
  • Dept clusters still look related

Cons

  • Still some agent overlap

Note: Option B (unique sprites) is the highest-quality outcome but the most design work. Worth it if you want CAST to feel like a real cast of characters.
